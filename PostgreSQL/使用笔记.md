## 安装

1、先创建data文件夹

2、初始化数据库，并设置数据库密码

```
"D:\Program Files\postgresql-18.1-2-windows-x64-binaries\pgsql\bin\initdb.exe" -D "D:\Program Files\postgresql-18.1-2-windows-x64-binaries\pgsql\data" -E UTF-8 --locale=chs -U postgres -W

注意："D:pgsql\data"  为数据路径
```

3、注册服务

使用 Windows 服务管理器来注册 PostgreSQL 服务

```
"<bin目录>\pg_ctl.exe" register -N PostgreSQL -D "<data目录>" -U WT -S auto -w -e "<data目录>\pg_log\pgstartup.log" -l "<data目录>\pg_log\pgsvc.log"


"D:\Program Files\postgresql-18.1-2-windows-x64-binaries\pgsql\bin\pg_ctl.exe" register -N postgresql-x64-18 -D "D:\Program Files\postgresql-18.1-2-windows-x64-binaries\pgsql\data"  -S auto -w -e "D:\Program Files\postgresql-18.1-2-windows-x64-binaries\pgsql\data\pg_log\pgstartup.log" -l "D:\Program Files\postgresql-18.1-2-windows-x64-binaries\pgsql\data\pg_log\pgsvc.log"

-N（服务名称）
-D（数据目录）
-U 指定运行 PostgreSQL 服务的 Windows 用户账户 ,指定会因登录无法启用
-S 指定服务的启动类型。auto 表示该服务将被设置为“自动”启动
-w 表示 pg_ctl 命令会等待服务注册完成
 -e 和 -l 参数来记录服务启动日志和通用日志

# 删除可能存在的服务
 pg_ctl unregister -N postgresql
# 下面是WIN系统服务管理命令
启动服务 **net start mysql**	net stop mysql(停止服务)  sc delete mysql删除服务
```

5、进程启动停止：

```
# 未注册服务的启动命令，服务会自己中断 
pg_ctl start -w -D "<data目录>" 启动pgsql进程
pg_ctl stop -w -D "<data目录>" 停止pgsql进程
"<bin目录>\pg_ctl.exe" -D "<data目录>" -l logfile start

pg_ctl stop -m smart
# SIGTERM "Smart"模式等待所有客户端断开连接以及任何在线备份结束。如果该服务器是热备，一旦所有的客户端已经断开连接，恢复和流复制将被终止。
pg_ctl stop -m fast
# SIGINT "Fast"模式（默认）不会等待客户端断开连接并且将终止进行中的在线备份。所有活动事务都被回滚并且客户端被强制断开连接，然后服务器被关闭。
pg_ctl stop -m immediate
# SIGQUIT "Immediate"模式将立刻中止所有服务器进程，而不是做一次干净的关闭。这将导致下一次重启时进行一次崩溃恢复。
```

## 清空表并重置自增ID

```
TRUNCATE TABLE ott_moth_data_20m RESTART IDENTITY
```

## 创建表

```
CREATE TABLE "ott_moth_data"."ott_moth_data_20m" (
  "id" bigserial   PRIMARY KEY,
  "时间" timestamp(6),

created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
;
```

## 添加时间字段，默认值为当前插入时间

```
-- 添加时间字段，默认值为当前插入时间
ALTER TABLE "OTT20米栅格特定APP数据"."场景与20米栅格空间信息关联"
ADD COLUMN IF NOT EXISTS "时间" TIMESTAMP DEFAULT now();
```

## 表添加自增主键ID

```
-- 给栅格空间信息表添加自增主键ID
ALTER TABLE "OTT20米栅格特定APP数据"."场景与20米栅格空间信息关联"
ADD COLUMN IF NOT EXISTS "id" BIGSERIAL PRIMARY KEY;
```

## 图层相交

```
SELECT p.*
FROM "规整数据_WIFI" p
JOIN "OTT数据处理图层"."OTT场景_all" pl ON p.geom&&pl.geom AND ST_Intersects(p.geom, pl.geom)
WHERE p.id>100 AND p.时间 > '2025-11-01' AND pl.场景 = '别墅';
```
